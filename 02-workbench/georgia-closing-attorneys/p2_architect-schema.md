# Architect Schema: Georgia Real Estate & Closing Attorneys

This document contains the raw schema and strategy output generated by the **Directory Intelligence (Architect)** Gem for the Georgia Closing Attorneys Directory (`01-GABAR`).

---

## LAYER 1 — Strategic Brief

### Section 1 — What This Directory Is
A statewide database-driven directory of real estate and closing attorneys where every attorney or law firm has its own rich, standalone page. This product operates as a high-utility directory database rather than a traditional content site to give users instantaneous access to verified legal credentials, local title agents, escrow services, and closing offices across all 159 Georgia counties.

#### Core Directory Metadata & Hierarchies
* **Primary Category Taxonomy:**
  * **By County:** Fulton, Gwinnett, Cobb, DeKalb, Chatham, Bibb, Richmond, Muscogee, etc.
  * **By City:** Atlanta, Savannah, Augusta, Macon, Columbus, Athens, Alpharetta, Marietta.
  * **By Service Specialties:** Residential Closings, Commercial Transactions, Investor-Friendly / Wholesaling, Double Closings, Probate Real Estate, Builder & Developer Services.
* **Primary Ingestion Sources:** State Bar of Georgia Public Member Directory, Google Places API.

---

### Section 2 — Why This Niche Wins on Google
This directory targets high-urgency, transactional local search intent with an explicit commercial purpose. Traditional search results are currently dominated by generic, broad-scope legal directories (like Avvo, Justia, or FindLaw) that lack specific real estate closing data fields (such as underwriter appointments, errors and omissions insurance limits, or mobile closing capabilities) and are easily outranked by a localized, hyper-focused programmatic directory filling this data gap. An example high-value search query is: *"investor friendly closing attorney Atlanta"*.

---

### Section 3 — Who Searches for This
A home buyer, real estate investor, lender, or real estate agent who is actively under contract on a property and needs to select a certified Georgia closing attorney immediately to handle title clearance and secure wire transfers. This directory captures a user in the high-stakes moment of a transaction rather than someone casually researching general legal principles.

---

### Section 4 — Revenue Model
* **Lever 1: Claimed Listings (MVP):** Closing operators and law firms pay a recurring monthly fee to claim their profile, upgrade to a premium design, unlock direct phone/website clicks, and strip out competitor advertisements.
* **Lever 2: Affiliate Placements (Phase 2):** Targeted native context placements for complementary services including residential property inspectors, home insurance providers, land surveyors, and structural engineers.
* **Lever 3: Lead Generation / RFQ Routing (Phase 2):** Unassigned quote requests or title review inquiries are captured via a centralized intake form and sold directly to verified local attorneys serving that specific county.
* **Lever 4: Email List (MVP):** High-converting lead magnet featuring the "Ultimate Georgia Home Buyer & Investor Closing Guide & Wire Fraud Checklist" to build a list of active real estate agents and high-volume investors.

#### Hormozi Reciprocity Playbook (Interactive Value)
* **Above-the-Fold Interactive Tool:** A free, un-gated **"Seller Net Sheet & Georgia Closing Cost Estimator"** that calculates county-by-county transfer tax ($1 per $1,000 of sales price), title fees, and closing fee estimates automatically.
* **Dynamic CTA Bridge:** Based on the county selected in the tool, a dynamic sidebar or carousel populates the top 3 verified local closing attorneys for that county with an instant CTA reading: *"Send Net Sheet to [Attorney Name] for a Free Title Consultation"*.

### SEO & AEO Flat Site Architecture

#### URL Structures
* **Pillar Page:** `domain.com/georgia-closing-attorney-requirements`
* **Explicit Geo-Page:** `domain.com/macon-real-estate-attorneys`
* **Listing Detail Page:** `domain.com/macon-john-doe-law`

#### Semantic Internal Linking Silo Structure
A flat, highly relevant contextual hierarchy ensures crawlability. The State hub page links out directly to the County landing hubs. Every County page links to its respective City pages and displays an indexed list of attorney child profiles located within that polygon. Breadcrumb paths link `Home ➔ County ➔ City ➔ Attorney Profile` with contextual reverse-linking enabled so that each profile passes authority back to its parent city and county geo-pages via anchor text like *"Best closing lawyers in Bibb County"*.

### Layer 4 — AI Enrichment Layer (44 fields)

#### 4a — listing_content (1 field)
* **`listing_content`**: A comprehensive, programmatic description containing between 200 and 400 words wrapped in HTML paragraph tags only. It covers exactly what services the listing offers, what transactional tools are present, step-by-step instructions on how to secure their closing escrow account, and an explicit trust signal verifying their length of tenure or operational scale under Georgia jurisdiction rules. No lists or markdown tags allowed.

#### 4b — Speakable Fields (3 fields)
> Each field contains exactly one paragraph of 2 to 4 sentences. The first sentence must directly and completely answer the question as if spoken out loud, completely avoiding starting with the listing firm name, list formats, or filler phrases.

* **`speakable_what_you_find`**: Answers *"What can I find / drop off / get at this listing?"*
* **`speakable_listing_details`**: Answers *"What is this listing and who runs it?"*
* **`speakable_quick_facts`**: Answers *"What are the key facts?"* (Must strictly start with: *"Key facts:"*)

#### 4c — Quick Facts Block (4 fields)
* **`quickfact_best_for`**: One short summary line starting with: *"This is the right place if you need to..."*
* **`quickfact_primary_items`**: Short text showing the top 3–4 practice areas.
* **`quickfact_fee_structure`**: Clear one-line standard fee summary.
* **`quickfact_access`**: Clear one-line accessibility summary (e.g., mobile closings available or in-office only).

#### 4d — Q&A Blocks (40 fields)

### Cialdini Persuasion & Monetization Stack
* **Loss Aversion Strategy:** All unclaimed profiles display a dynamic system warning banner stating: *"Unverified Attorney Listing — Competitor advertisements may be dynamically generated on this profile."* Directly below the banner, a widget features active verified alternatives in the same zip code. Legally verifying and claiming the listing instantly deletes all competitor ads ("Conquest Ads") and unlocks direct phone dialing links.
* **Featured Tier Specifications:** Subscribing premium firms obtain pinned visibility at the top of their target city/county geo-archives, high-contrast listing cards with a "Verified closing expert" badge, direct routing for email/SMS inquiries, and a guaranteed placement inside the "Seller Net Sheet" calculator CTA widget.
* **Lead Routing (RFQ Form):** A structured multi-step quote request system tracks key deal points (e.g., loan file type, purchase price, target date, clear title hurdles). Completed inquiries are parsed programmatically and instantly dispatched via secure hooks directly to premium or verified operators covering that specific target region.

### Anti-Thin-Content Gate
Run after enrichment, never before. Listings that fail are held as drafts.
1. `listing_content` must contain at least 100 words (strip HTML tags before evaluation).
2. The primary Layer 3 field (`bar_license_number`) is verified not empty.
3. A minimum of 5 out of the 7 Phase 1 programmatic Q&A block fields are populated with text of 20 or more words each.
4. The `listing_img` destination path holds a verified, non-empty text string.
5. `listing_name` and `street_address` are both present and structural.

---

### Pipeline Warnings
1. **Enable both Google APIs separately.** The Places API and the Maps Static API must be turned on individually within your Google Cloud Console workspace. Enabling one does not activate the other.
2. **Remove all "skip if already enriched" logic.** This configuration will silently drop records from earlier pipeline attempts. Always run extraction directly from the clean original seed source.
3. **Run the anti-thin gate after enrichment.** Programmatic AI fields do not structurally exist until the validation pipeline completes. Evaluating the gate beforehand results in complete file failure.
4. **Use two separate coordinate fields — manual_lat and manual_lng.** Merging geographic points into a single coordinate container field silently breaks front-end mapping canvas renders.
5. **Test with 5 records before full import.** Standard directory structural plugins save custom variables using database-side alphanumeric hashes rather than human-readable text label names. Pull actual database keys via a structural test sample before executing full database synchronization.
